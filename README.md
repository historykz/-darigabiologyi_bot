# 🎓 Образовательный Telegram-бот

Бот для управления учебным процессом: **Администратор**, **Кураторы**, **Ученики**.
Сдача рабочих тетрадей фото → PDF, дедлайны с автонапоминаниями, отчёты, резервные копии.

Стек: **aiogram 3.x + PostgreSQL (SQLAlchemy async) + APScheduler**. Деплой: **GitHub → Railway**.

---

## 🚀 Деплой на Railway (по шагам)

### 1. Залить код на GitHub
```bash
cd bot
git init
git add .
git commit -m "educational telegram bot"
git branch -M main
git remote add origin https://github.com/ВАШ_АККАУНТ/ВАШ_РЕПО.git
git push -u origin main
```

### 2. Создать проект на Railway
1. Зайти на railway.app → **New Project** → **Deploy from GitHub repo** → выбрать репозиторий.
2. Railway сам определит Python и поставит зависимости из `requirements.txt`.

### 3. Подключить базу PostgreSQL
- В проекте: **New** → **Database** → **Add PostgreSQL**.
- Railway автоматически создаст переменную `DATABASE_URL`. Трогать её не нужно.

### 4. Прописать переменные окружения
В сервисе бота → вкладка **Variables** → добавить:

| Переменная   | Значение                                  |
|--------------|-------------------------------------------|
| `BOT_TOKEN`  | токен от @BotFather                       |
| `ADMIN_ID`   | ваш числовой Telegram ID (узнать у @userinfobot) |

`DATABASE_URL` уже задан Railway автоматически.

### 5. Запуск
Railway запускает `python main.py` (из `Procfile` / `railway.toml`).
При первом старте бот **сам создаёт все таблицы** — Alembic для старта не нужен.

Готово. Откройте бота в Telegram и напишите `/start`.

## 🔑 Как назначить кураторов и учеников

1. **Вы (админ)** пишете `/start` → видите меню администратора.
2. **👨‍🏫 Кураторы** → *Назначить куратора* → ввести `@username` или Telegram ID.
   - Если ввели ID — куратору сразу придёт уведомление.
   - Если `@username` — роль активируется, когда он напишет `/start`.
3. Куратор пишет `/start`, создаёт группы (**📂 Мои группы**) и добавляет учеников (**➕ Добавить**).
4. Ученик пишет `/start` — если куратор добавил его по Telegram ID, он сразу попадает в свою группу.

> ⚠️ Чтобы ученик распознавался автоматически, лучше добавлять его по **Telegram ID**.
> При добавлении по `@username` сопоставление произойдёт, когда ученик впервые напишет боту
> (username сохраняется в его профиле).

## 💾 Резервные копии

- **💾 Резервная копия → 📤 Скачать сейчас** — выгрузка JSON или Excel.
- **🔄 Восстановить из файла** — после обновления бота загрузите `backup_*.json`, данные вернутся (upsert).
- **Автобэкап** — каждое воскресенье 03:00 по Астане JSON приходит админу в личку.

## 📊 Google Sheets (опционально)

1. Создать service-аккаунт в Google Cloud, скачать `service_account.json`.
2. Добавить файл в репозиторий **или** в Railway Variables путь `GOOGLE_CREDS_FILE`.
3. Дать service-аккаунту доступ к таблице (его email — «Редактор»).
4. В боте: **📊 Google Sheets** → прислать ссылку на таблицу.

Без настройки эта функция просто отключена и ни на что не влияет.

## 🗂 Структура проекта

```
bot/
├── main.py                 # запуск, регистрация роутеров, планировщик
├── config.py               # переменные окружения, часовой пояс
├── database/               # модели, CRUD, сессия
├── handlers/               # admin / curator / student / common + фильтр роли
├── services/               # PDF, роли, уведомления, планировщик, бэкап, restore, sheets
├── keyboards/              # клавиатуры по ролям
├── states/                 # FSM-состояния
├── requirements.txt
├── Procfile
├── railway.toml
└── .env.example
```

## 🧪 Локальный запуск
```bash
pip install -r requirements.txt
cp .env.example .env   # вписать BOT_TOKEN, ADMIN_ID (DATABASE_URL можно оставить sqlite)
python main.py
```
Без `DATABASE_URL` бот использует локальный SQLite (`bot.db`) — удобно для теста.

## ⏰ Часовой пояс
Всё хранится в **UTC**, отображается по **Астане (UTC+5)**. Время дедлайна вводится тоже по Астане.
